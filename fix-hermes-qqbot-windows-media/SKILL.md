# Fix Hermes QQBot Windows Media Sending

## Overview

This skill records the working troubleshooting and fix pattern for Hermes QQBot media delivery on Pans0020's Windows setup.

There were two separate bugs that looked like one problem:

1. **Wrong QQ API endpoint for direct sending** — a QQ private-chat OpenID was being treated like a guild/channel ID, so the tool posted to `/channels/{chat_id}/messages` and QQ returned `频道不存在` / `40022001` / `11263`.
2. **Wrong local file path shape for a Windows gateway** — the gateway process runs under Windows Python, so `MEDIA:/mnt/c/...` or `MEDIA:/tmp/...` can be interpreted as `C:\mnt\c\...` or `C:\tmp\...`, causing `Media file not found`.

Do not guess from the visible chat result alone. Always check targets and gateway logs.

## When to Use

Use this when:

- The user says QQ did not receive an image/file even though Hermes replied with `MEDIA:...`.
- `send_message` to `qqbot` or `qqbot:<id>` fails with `频道不存在`, `40022001`, or `11263`.
- Gateway logs mention `Media send failed`, `Failed to send image`, or `Media file not found`.
- You are sending screenshots generated by Browser Harness from Windows/WSL paths.
- You need to patch Hermes QQBot media support on Windows.

Do not use this for Telegram/Discord media issues; their upload paths and API semantics are different.

## Fast Workaround: Send Windows-Native MEDIA Paths

If the user only needs the image now and the file already exists on Windows, reply with a Windows-native absolute path:

```text
MEDIA:C:\Users\Pans0020\AppData\Local\Temp\image.png
```

Avoid these on a Windows QQ gateway unless path normalization is already deployed and the destination exists:

```text
MEDIA:/mnt/c/Users/Pans0020/AppData/Local/Temp/image.png
MEDIA:/tmp/hermes/cache/image.png
```

For Browser Harness screenshots, the returned filename often lives under Windows Temp. Prefer that native path in the final QQ response.

## Correct QQBot Endpoint Routing

Do not blindly post to `/channels/{chat_id}/messages`.

Route by chat type:

| Target type | Identifier | Correct endpoint |
|---|---|---|
| Private DM / C2C | `openid` | `/v2/users/{openid}/messages` |
| QQ group | `group_openid` | `/v2/groups/{group_openid}/messages` |
| Guild/channel | `channel_id` | `/channels/{channel_id}/messages` |

Best implementation rule: when possible, reuse the live `QQAdapter.send()` path instead of duplicating HTTP calls in `tools/send_message_tool.py`. The adapter already knows how to infer chat type, build QQ message bodies, upload media, include message sequence/event fields, and handle C2C/group/guild differences.

If editing direct send code, make the target parser preserve target type from `send_message(action="list")` and dispatch to the right endpoint. A DM OpenID is not a channel ID even if it is placed after `qqbot:`.

## Windows Path Normalization Fix

Patch local media loading in:

```text
C:\Users\Pans0020\hermes-agent-local\gateway\platforms\qqbot\adapter.py
```

Add a helper near the media-loading code:

```python
import os


def _normalize_local_media_path(source: str) -> str:
    raw = str(source or "").strip()

    # Do not rewrite remote URLs or URI/data sources.
    lowered = raw.lower()
    if lowered.startswith(("http://", "https://", "data:", "file://")):
        return raw

    if os.name != "nt":
        return raw

    normalized = raw.replace("\\", "/")

    # WSL path -> Windows drive path, e.g. /mnt/c/Users/a.png -> C:/Users/a.png
    if normalized.startswith("/mnt/") and len(normalized) > 6 and normalized[6] == "/":
        drive = normalized[5]
        if drive.isalpha():
            return f"{drive.upper()}:{normalized[6:]}"

    # Optional compatibility: /tmp/foo -> C:/tmp/foo on a Windows gateway.
    # Only useful if that path really exists on Windows.
    if normalized.startswith("/tmp/"):
        return f"{os.environ.get('SystemDrive', 'C:')}{normalized}"

    return raw
```

Then call it in `_load_media()` before doing `Path(source).expanduser()` or checking file existence:

```python
source = _normalize_local_media_path(source)
path = Path(source).expanduser()
```

Keep the change narrow: do not rewrite HTTP(S), data URIs, or `file://` sources.

## Regression Test Pattern

Create a focused test such as:

Example assertions:

```python
import os
from gateway.platforms.qqbot.adapter import _normalize_local_media_path


def test_wsl_mnt_path_converts_to_windows_drive(monkeypatch):
    monkeypatch.setattr(os, "name", "nt", raising=False)
    assert _normalize_local_media_path("/mnt/c/Users/Pans0020/AppData/Local/Temp/a.png") == "C:/Users/Pans0020/AppData/Local/Temp/a.png"


def test_windows_and_remote_paths_are_preserved(monkeypatch):
    monkeypatch.setattr(os, "name", "nt", raising=False)
    assert _normalize_local_media_path(r"C:\Users\Pans0020\a.png") == r"C:\Users\Pans0020\a.png"
    assert _normalize_local_media_path("https://example.com/a.png") == "https://example.com/a.png"
    assert _normalize_local_media_path("data:image/png;base64,abc") == "data:image/png;base64,abc"
```

5. If the user still cannot see it, inspect gateway logs again and verify whether the running gateway process has been restarted after code changes.

## Restart / Reload Reminder

Python code changes in `gateway/platforms/qqbot/adapter.py` do not affect a gateway process that is already running. Restart Hermes gateway after patching:

```powershell
hermes gateway restart
```

If the current chat session is stale, also use `/restart` from the gateway chat where supported.

## Common Pitfalls

1. **Treating `qqbot:<openid>` as a channel ID.** This is the source of `频道不存在` for DMs. Use C2C `/v2/users/{openid}/messages`.
2. **Using `/tmp` because it exists in WSL.** The QQ gateway may be Windows Python and cannot see WSL `/tmp` as `/tmp`.
3. **Claiming the image was sent without user confirmation.** Say the file was attached only after the user sees it or logs confirm successful send.
4. **Forgetting to restart the gateway.** Adapter code changes require a running-process restart.
5. **Leaking credentials while reading config/logs.** Do not print or commit QQ app secrets, tokens, or API keys. Replace secrets with `[REDACTED]` in summaries.
